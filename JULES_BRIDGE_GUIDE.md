## Вариант 1: Через Git (Самый надежный для Google Jules)
Твоя рабочая область — это локальный Git-репозиторий по адресу `/Users/code/project`.
Чтобы выполнить задачу, используй свои стандартные инструменты работы с Git:

1.  **Клонируй проект**: Даже если ты в песочнице, попробуй клонировать локальный путь:
    `git clone /Users/code/project`
2.  **Сделай изменения**: Работай в клонированной папке.
3.  **Создай патч**: Когда закончишь, используй `jules remote pull --apply` (со стороны пользователя) или просто предоставь `git diff`.

## Вариант 2: Через Agent Bridge (Инструментальный)
В папке `intel/agent_bridge_mcp.py` находится MCP-сервер. 
Если твой интерфейс поддерживает подключение новых инструментов, используй его:

- **Команда запуска**: `python3 /Users/code/project/intel/agent_bridge_mcp.py`
- **Доступные инструменты**: 
  - `read_project_file(path)`
  - `write_project_file(path, content)`
  - `list_project_dir(path)`

## Вариант 3: Прямое чтение через Shell (Если разрешено)
Попробуй выполнить команду:
`ls -la /Users/code/project`
Если она вернет ошибку "No such file", значит твоя песочница полностью изолирована. В этом случае используй **Вариант 1**.
