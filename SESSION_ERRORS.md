# Ошибки сессии 2026-04-02 — чтобы не повторять

## 1. Автоматизация формы через Kapture
**Ошибка:** Попытка автозаполнения React-dropdown через fill() — не работает.
**Причина:** React-select требует симуляции событий keydown/change, fill() не триггерит React state.
**Решение:** Для React-форм использовать javascript_tool (если доступен) или давать пользователю готовый гайд.

## 2. Вкладка браузера не в фокусе (visibility: hidden)
**Ошибка:** Screenshot и click не работают когда вкладка hidden.
**Причина:** Kapture требует активной вкладки. Показ через show() не всегда переключает фокус.
**Решение:** Использовать show() + небольшая пауза, или просить пользователя кликнуть на вкладку.

## 3. Фоновые bash-команды с &
**Ошибка:** Запуск gemini с & — вывод терялся, файл не писался.
**Причина:** Background процесс не захватывается в task output.
**Решение:** Использовать run_in_background: true в Bash tool ИЛИ запускать синхронно.

## 4. Субагент без сетевых прав
**Ошибка:** Agent tool запущен для проверки дублей — все сетевые инструменты заблокированы.
**Причина:** Субагенты по умолчанию имеют ограниченные права на сеть.
**Решение:** Для веб-задач использовать WebFetch/WebSearch напрямую в главном контексте.

## 5. Qwen через ollama — медленно
**Проблема:** 14b модель работает ~3-5 минут на большой промпт.
**Решение:** Для быстрых задач использовать qwen2.5-coder:7b или gemini.

## 6. Неправильная цель (AnythingLLM)
**Ошибка:** Потрачено время на AnythingLLM — хорошо защищён.
**Урок:** Приоритизировать: звёзды < 10k, Python/Node с HTTP UI, file upload.

- ollama API через curl — прямой вызов qwen ✅
- Agent tool для анализа кода и написания отчётов ✅
- WebFetch для проверки дублей ✅
- JSON-протокол в intel/results/ ✅
